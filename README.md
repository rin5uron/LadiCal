# LadiCal

完全ローカルで動作する、iOS向けの軽量パーソナルログアプリ。

元になったのは Android アプリ「LadiCal」の使い勝手だったが、サポート終了とデータ消失の不安をきっかけに、iOS で長く使える自前アプリとして再設計する。

コンセプトは以下の4点。

- 軽い
- 速い
- 消えない
- 見られない

## このプロジェクトについて

Android アプリ「LadiCal」を長く使っていたが、サポート終了とデータ消失の不安があったため、iOS で使い続けられる個人用ログアプリとして再構築する。

目標は、体調管理に閉じない「長く使えるパーソナルログツール」を作ること。

## 目的

- 体調や気分をカレンダー中心に記録できる
- 絵文字・トグル・メモで素早く入力できる
- 用途を限定せず、個人ごとに自由なログとして使える
- オフライン前提で、安全に長く記録を残せる

## 基本方針

- オフライン完結を最優先にする
- UI は最小構成にする
- 入力は 1〜2 タップで完結できることを目指す
- 振り返りしやすい情報設計にする
- 設定やカスタム項目で、ユーザーごとに使い方を変えられるようにする

## 技術選定理由

このアプリは iPhone を最優先に作るため、iOS ネイティブの標準構成を採用する。

- `SwiftUI`
  - iOS 向け UI を素直に実装しやすい
  - AI と対話しながら画面単位で改善しやすい
- `Core Data`
  - ローカル保存中心の設計と相性がよい
  - 日付単位の記録管理に向いている
- `LocalAuthentication`
  - 将来の Face ID / パスコード対応に使う
- `PhotosUI`
  - 写真ライブラリから画像を選択するために使う

現時点では、iPhone で確実に使えるものを作ることを優先し、Android 展開は将来課題として扱う。

## MVP 範囲

最初の実装では、以下を優先する。

1. 月カレンダー UI
2. 日付選択
3. 1日ごとの記録保存
4. 絵文字入力
5. メモ入力
6. 画像保存と表示
7. 保存済みリスト表示
8. カスタム項目の保存
9. CSV エクスポート
10. ローカル自動バックアップ

以下は後続で段階追加する。

- 基礎体温
- パスコード / Face ID
- iCloud バックアップ
- ヘルスケア連携
- 妊娠モード

## 設計のポイント

- 既存アプリの弱点だったデータ消失不安を、ローカル保存とバックアップ前提で解消しようとしている点
- 絵文字と ON / OFF 項目を同じ `CustomItem` として扱う、拡張しやすいデータ設計
- 1 日 1 レコードを基本にして、初心者でも破綻しにくい MVP に絞っている点
- 周期管理や妊娠モードなど重い機能を後段に分け、先に使える核を作る方針
